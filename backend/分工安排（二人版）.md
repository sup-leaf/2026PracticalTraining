|------|------|------|
| 6.1 | 实习入职报备 | `POST /api/internship/start` —— 学生确认入职，关联投递记录（job_id → delivery_id），记录入职日期、企业联系人 |
| 6.2 | 实习日志提交 | `POST /api/internship/log` —— 学生按周提交实习日志（文本 + 可选图片/附件），`GET /api/internship/logs` 查看历史 |

## 三、成员二：马勇

> 已完成科研项目选人模块（发布→申请→审核模式）和 Knife4j 文档，后续竞赛组队模块复用完全相同的三层结构，代码迁移成本低。

### P1 任务（优先）

#### 竞赛组队模块（编号 5.1～5.4）

| 编号 | 事项 | 说明 |
|------|------|------|
| 5.1 | 发布组队招募 | `POST /api/competition/publish` —— 队长（学生）发布组队帖：赛事名称、缺人岗位（如 UI 设计、算法）、技能要求、人数上限、截止日期。新表 `t_competition_team` |
| 5.2 | 标签筛选匹配 | `GET /api/competition/list?keyword=&skillTag=&major=` —— 按技能标签/专业检索组队帖。后端用 LIKE 或 FIND_IN_SET 匹配 `skill_tags` 字段 |
| 5.3 | 学生申请入队/队长审核 | `POST /api/competition/apply` —— 学生申请；`POST /api/competition/audit` —— 队长（发布者）通过/拒绝。新表 `t_team_application` |
| 5.4 | 组队状态管理 | 队长可操作：结束招募（status: 1→2）或重新开放（2→1）；满员自动关闭；`GET /api/competition/my` 查看我发布/我加入的队伍 |

- **新建文件**：`entity/CompetitionTeam.java`、`entity/TeamApplication.java`、`mapper/CompetitionTeamMapper.java`、`mapper/TeamApplicationMapper.java`、`service/CompetitionService.java`、`service/impl/CompetitionServiceImpl.java`、`controller/CompetitionController.java`、`sql/competition.sql`
- **前端配套**：新建"竞赛组队"页面（队长发帖 + 学生浏览申请 + 队长审核）

### P2 任务（延后）

#### 2. 实名认证（编号 2.6）

- **目标**：学生上传校园卡照片，管理员审核通过后标记为已认证
- **实现**：`POST /api/auth/verify` 上传照片 → 管理员 `PUT /api/admin/verify/{userId}?status=1`。用户表新增 `verified` 字段
- **影响文件**：`AuthController.java`、`UserServiceImpl.java`

#### 3. 数据脱敏返回（编号 2.7）

- **目标**：API 返回的手机号、邮箱部分隐藏（如 `138****1234`、`z***@bjtu.edu.cn`）
- **实现**：新建 `util/DesensitizeUtil.java` 工具类，在 `Result` 序列化前或 Controller 返回前对敏感字段处理
- **影响文件**：`DeliveryServiceImpl.java`、`ResumeServiceImpl.java` 的返回数据构造处

#### 4. 短信验证码（编号 9.1）

- **目标**：注册/找回密码时发送短信验证码
- **实现**：对接阿里云/腾讯云短信 SDK，新建 `util/SmsUtil.java`，Redis 存储验证码（5 分钟过期），`POST /api/auth/sms/send` 和 `POST /api/auth/sms/verify`
- **影响文件**：`pom.xml`、`application.yml`、`util/SmsUtil.java`

#### 5. 系统公告/通知（编号 9.2）

- **目标**：管理员发布全站公告，学生/企业首页可见
- **实现**：新表 `t_announcement`，`POST /api/announcement/publish`（管理员），`GET /api/announcement/list`（全员）
- **新建文件**：`entity/Announcement.java`、`mapper/AnnouncementMapper.java`、`service/AnnouncementService.java`、`service/impl/AnnouncementServiceImpl.java`、`controller/AnnouncementController.java`、`sql/announcement.sql`

#### 6. 定时任务（编号 9.3）

- **目标**：自动化运维
- **实现**：新建 `task/ScheduledTask.java`，使用 Spring `@Scheduled`
  - 每日 0 点：过期岗位自动下架（`t_job.status: 1→0`，截止日期已过）
  - 每日 8 点：实习周期到期提醒（查询 `t_internship`，距结束 < 7 天）
  - 每日 2 点：数据统计归档（汇总昨日投递/注册数据到统计表）

## 五、文件冲突规避

两人负责的模块独立，**新建文件零交集**（entity/mapper/service/controller/sql 各自命名空间不同）。

| 可能重叠处 | 解决方式 |
|-----------|---------|
| `pom.xml` | 两人各加一条依赖，先后提交，Git 自动合并 |
| `application.yml` | 各自加在文件末尾，不修改对方配置项 |
| `进度管理.md` | 各自更新自己的编号行，不改动对方内容 |
