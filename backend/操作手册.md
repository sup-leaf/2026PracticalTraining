## 一、环境与配置

| 组件 | 版本 | 端口 |
|------|------|------|
| JDK | 17+ | - |
| Maven | 3.6+ | - |

### 你需要改的配置

打开 `src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    password: 你的MySQL密码   # ← 必须改

ai:
  dashscope:
    api-key: 你的阿里DashScope API Key # ← AI优化必改
```

**其他配置项说明：**

| 配置项 | 默认值 | 含义 |
|--------|--------|------|
| `server.port` | `8080` | 后端端口 |
| `upload.path` | `./uploads` | **相对路径**，相对于启动命令所在目录（即 `backend/` 目录下）。Spring Boot 会自动创建。换机器无需修改。 |
| `spring.redis.host` | `localhost` | Redis 地址 |

## 三、配置项详解

### upload.path 路径机制

```
application.yml:
  upload:
    path: ./uploads
```

- `./uploads` 是**相对于启动命令所在目录**的路径
- 在 `backend/` 目录下执行 `mvn spring-boot:run`，实际路径就是 `backend/uploads/`
- 换机器不需要修改，Spring Boot 启动时自动创建目录
- 文件通过 `/uploads/2026/05/16/xxx.pdf` URL 访问
- `WebConfig.java` 将 `/uploads/**` URL 映射到本地文件系统
- 前端通过 Vite 代理 `'/uploads': { target: 'http://localhost:8080' }` 访问

### 数据库密码

前端发送的密码在浏览器端做了 MD5 → 后端收到后再做一次 MD5 存入数据库。这是双重 MD5。

**计算方式：**
```
数据库存储值 = MD5( 浏览器发来的MD5值 )
             = MD5( MD5(明文密码) )
```

## 四、完整验证流程（6 步走通闭环）

### 账号信息速查

| 账号 | 密码 | 类型 | 来源 |
|------|------|------|------|
| admin | 123456 | 教师 | 数据库手动插入（见第二章步骤2） |
| 企业账号 | 任意 | 企业 | 前端注册（注册后待审核，教师通过后才可登录） |
| 学生账号 | 任意 | 学生 | 前端注册（即时可用） |

### 步骤

```
1. 教师登录 admin/123456
       ↓
2. 退出 → 注册一个企业账号 → 重新用教师登录 → 企业管理→企业入驻审核 → 点"通过"
       ↓
3. 退出 → 用通过的企业账号登录 → 岗位中心→发布岗位 → 填写信息发布
       ↓
4. 退出 → 注册学生账号 → 登录 → 我的简历（填写+上传文件+保存） → 岗位管理 → 点"投递"
       ↓
5. 退出 → 企业账号登录 → 投递中心→投递管理 → 看到投递 → 展开看详情 → 点"已查看" → 点"面试中"(填备注) → 点"录用"
        ↓
6. 退出 → 学生账号登录 → 实习管理 → 看到已录用投递 → 点"发起实习" → 提交日志
        ↓
7. 退出 → 企业账号登录 → 实习管理→实习生列表 → 点"评价" → 打分+评语
        ↓
8. 教师登录 → 数据看板→人才供需大屏 → 查看实习率/趋势/排行
```

## 五、常见问题

### 端口被占用
```powershell
netstat -ano | findstr 8080
taskkill /PID <PID> /F
```

### 数据库连接失败
1. 确认 MySQL 服务已启动
2. 确认 `application.yml` 中密码为你自己的
3. 确认数据库 `campus_market` 已创建

### 文件上传失败 / 返回 401
- `el-upload` 需携带 JWT
- 确认已登录且 token 未过期

### 文件链接点不开（404）
- Vite 代理需配置 `/uploads` → `http://localhost:8080`（已在 `vite.config.js` 中）
- 重启 `npm run dev` 让代理生效

### 企业注册后无法登录
- 企业注册后 status=0（待审核），需教师审核通过后 status=1 才能登录

### 上传的目录在哪里
- `upload.path: ./uploads` 相对路径下的 `uploads/2026/05/16/` 目录
- 自动按日期创建子目录

### 实习相关
- 学生只有被"已录用"后才能发起实习
- 一个学生同时只能有一个进行中实习
- 结束后可发起新实习
- 企业评分后学生端"已通过"处可见

## 六、接口测试（curl / Postman 示例）

```bash
# 1. 学生登录，获取 token
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"stu01","password":"<MD5值>","userType":1}'
# 响应: {"code":200,"data":{"token":"eyJ...","user":{...}}}

# 2. 带 token 查岗位（替换 YOUR_TOKEN）
curl http://localhost:8080/api/job/list?page=1&size=10 \
  -H "Authorization: Bearer YOUR_TOKEN"

# 3. 投递
curl -X POST "http://localhost:8080/api/delivery/apply?jobId=1" \
  -H "Authorization: Bearer YOUR_TOKEN"

# 4. 查看我收到的投递（企业/教师）
curl http://localhost:8080/api/delivery/publisher?page=1&size=10 \
  -H "Authorization: Bearer YOUR_TOKEN"

# 5. 更新投递状态
curl -X PUT "http://localhost:8080/api/delivery/status?deliveryId=1&deliveryStatus=3&note=录用" \
  -H "Authorization: Bearer YOUR_TOKEN"

# 6. 学生发起实习（用已录用投递）
curl -X POST "http://localhost:8080/api/internship/start?deliveryId=1" \

  -H "Authorization: Bearer YOUR_TOKEN"
```

## 七、状态码速查

| 字段 | 值 | 含义 |
|------|----|------|
| userType | 1 / 2 / 3 | 学生 / 企业 / 教师 |
| user status | 0 / 1 / 2 | 待审核 / 正常 / 已拒绝（仅企业） |
| jobType | 1 / 2 / 3 | 实习 / 全职 / 科研助理 |
| delivery status | 0→1→2→3↔4 | 待查看→已查看→面试中→已录用(可撤回)/已拒绝(可重开) |
| coop type | 1 / 2 | 深度合作 / 校外普通 |
| memberLevel | 0 / 1 | 免费 / VIP |
| internship status | 0 / 1 / 2 | 进行中 / 已完成 / 提前终止 |
| Result.code | 200 | 成功 |
| Result.code | 401 | 未登录/token 过期 |
| Result.code | 403 | 无权限访问 |
| Result.code | 500 | 业务错误 |
